How the Web Works
The web is a bunch of computers that communicate with each other. When a person goes to a web page like www.google.com, their computer sends a HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. This video does a good job of explaining more.
HTML
HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks". This video gives a good overview.

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. This video explains the distinction well.

Computers are stupid because they interpret instructions literally. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

The "tree-like structure" of HTML
The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on... In a browser, this structure shows up as a series of nested boxes. There are oxes inside of boxes inside of boxes, and so on...
The relationship between indented HTML and boxes
When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

Variables
Variables in Python and other computer codes are similar to any type of variable - the variable represents something else, so we can easily change that value. It also helps make code much easier to read.
Variable Assignment
The process of giving a variable a new value is called "Assignment." This is done by entering the name of the variable, an equals (=) sign, and then the desired value.
Equals Sign Clarification
Traditionally, we think of (=) to mean "equals" as in math, like 2+3=5. In Python, = is more like an arrow (pointing from right to left) showing that the value on the right side will be assigned to the name on the left side.
Common Uses of Variables
Variables can be useful when there is a good chance you will change a value. Variables (should) make it easier to understand the function of the code. Variables can be used to reference other values to make very complex calculations and perform very complex tasks.
What is the difference between 2+2 and "2"+"2"?
In Python, print 2+2 will show the value 4, while print "2"+"2" will show the value 22. In the first example, Python does the math, in the second, it concatenates the two strings, showing a result of 22.
Functions
A Function or Procedure is an operation or task that a computer can be programmed to do. Important Distinction - Making a function involved defining what the function does, essentially telling the computer what it will do when you eventually call it to perform the function. Using a function is the act of calling the computer to perform it. Functions can help a programmer avoid repitition by making tedious tasks automated. For example, instead of creating a unique user ID for a customer manually, a progrmamer can write a function which automatically takes a few inputs (ie name, ss#, etc) and creates a unique identifier based on the parameters of the function. The return keyword is used to tell Python what the output of the function is. Without it, the function will not know what you are asking for when you call the function.
Structured Data
Data that is placed into related fields, or sorted in some manner. ie, this string '24 35 75 93 745 02 787 monday tuesday wednesday thursday friday saturday sunday' is technically data, but because it has no structure, it is not very useful. [[monday, 24], [tuesday, 35], [wednesday, 75], [thursday, 93], [friday, 745], [saturday, 02], [sunday, 787] has more structure, and is more powerful and more useful.
Mutability
Strings and Lists behave differently. When you've assigned a string to a variable, and then change the string in that variable, you do not actually alter the string itself. Instead, you make an entirely new string, and reassign the variable to the new string. Strings are therefore 'imutable.' Lists, however, are 'mutable.' Rather than creating a new list and reassigning the variable to it, you can change individual elements within the list, which would affect all the variables which refer to the list.
difference between APPEND and +
Append adds a new element to the end of the list (changes the original list) whereas + creates a new list, and concatenates the original list and the new element or list into 1 new list.
Lists
Lists are a powerful way of structuring data. Unlike strings, which can only contain characters, lists can contain any element, including other lists. Nested lists are a great way of storing and accessing related data.

Object Oriented Programming
OOP is a type of programming which allows the programmer to reuse code for other projects. Rather than only using functions to perform tasks/actions, the programmer can create and use "classes," which can apply to many similar objects. With large projects, this can save the programmer time writing the code, and also make the code more efficient and readable for other people.
Objects
Like in the physical world, Objects are items which can be used and manipulated by the programmer. For instance, in our movie website example, each piece of info related to the movies, such as the youtube url, poster url, description, and more, could be considered objects. Programmers can use code to manipulate those objects.
Class
Classes are like templates - often times, while objects are not exactly the same, they are similar in many ways. For example, while Avatar and School of Rock are very different movies, they are both movies. Each movie is an instance (to be explained soon) of the larger category, or "class," movies. In code, it's like a blueprint. While a blueprint can give info like the number of rooms, or size of the walls in a building, the same blueprint can be used to create different buildings.
Instance
An "Instance" is 1 example of a class. As mentioned earlier, Avatar and School of Rock are both instances of the "movie" class. In real life, an individual person (like myself, Andy from Udacity, or any other person) would be an instance of the larger class, Humans.
Module
A module is software that can be downloaded and used on various systems to perform complex tasks which would otherwise be much more difficult. For example, we use the Twilio module in order to send text messages to phones, using our codes.
Method
Methods are functions that are parts of specific classes or objects. In every class, there are various methods, the most universal being __init__, which are bound to those classes or objects. Whenever a programmer calls a class, the __init__ method is called to initialize the class, allowing the programmer to utilize the class.
Library
A Library is essentially a storage facility for information. There is far too much computer programming info for humans to completely remember, so we store the info in libraries (such as the Python Standard Library) in order to easily access that info when it is needed.
Class Variables
Class Variables are similar to the variables we've studied earlier in the course (called instance variables) but the main difference is that there is only one copy of a Class Variable which is stored in the class and shared accross all instances of that class.
Inheritance
Much like children inherit many characteristics and possesions from their parents, certain classes (aptly called children classes) can inherit characteristics, such as methods, from other classes (the parent). Unless overridden, (see Method Overriding, next), a child class will inherit all the methods and characteristics of its parent class. When defining the child class, simply place the name of the parent class as an argument to the child. This allows programmers to make similar classes without copying and pasting, or retyping, large amounts of code for just a few small changes.
Method Overriding
While Child Classes automatically inherit everything from its parent, programmers need to be able to override some of the methods, in order to make a class which is similar, but not the same. Simply make the change within the class definition, and it will override the methods it has inherited from its parent class.
